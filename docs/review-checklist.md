# レビューチェックリスト

コードレビュー（自己レビュー含む）のときに確認する観点をまとめます。
PR を出す前、またはレビューするときに上から順に確認してください。

## 1. 動作・検証

- [ ] `cargo build` が成功する。
- [ ] `cargo test` が成功する（テストがある場合）。
- [ ] `cargo fmt` 済みで、差分に整形漏れがない。
- [ ] `cargo clippy` の警告が増えていない。

## 2. アーキテクチャ / モジュール

- [ ] 変更が適切なレイヤー（`model` / `presentation`）に置かれている。
- [ ] `main.rs` に詳細な処理を書き込んでいない（入口を呼ぶだけになっている）。
- [ ] 新しいディレクトリには `mod.rs` を置き、`pub mod` で公開している。
- [ ] 不要に `pub` を増やしていない（公開は最小限）。

## 3. コメント / ドキュメント

- [ ] コメントが日本語で書かれている。
- [ ] 既存の日本語コメントを不必要に英語化していない。
- [ ] `src/model/*.rs` は `src/model/model.md` の規約に従っている
      （`//!` モジュールコメント、構造体・各フィールドの `///` コメント）。
- [ ] 設計や仕様を変えた場合、関連する `docs/` を更新している。

## 4. 変更範囲

- [ ] 変更がリクエストの対象に絞られている（無関係な変更が混ざっていない）。
- [ ] 明らかな誤字・表記ゆれ（例: 「現す」→「表す」）を残していない。

## 5. Git / PR

- [ ] `main` ブランチに直接コミットしていない（作業ブランチで作業した）。
- [ ] コミットメッセージが変更内容を簡潔に説明している。
