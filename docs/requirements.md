# 要件

このドキュメントは「何を作るか」を記載します。

> 注記: 「現状（実装済み）」は現在のコードに基づく事実です。
> 「今後の候補」はまだ確定していない草案です。優先度や採否はレビューで決めてください。

## 1. 目的

- ターミナル上で動作する、シンプルな CLI の Todo アプリケーションを作る。
- 学習・教育を目的とし、各ファイルに手厚い日本語コメントを付けながら、
  レイヤー分割（`model` / `presentation`）の設計を学ぶ題材とする。

## 2. 利用者・利用シーン

- 利用者: コマンドライン操作ができる開発者・学習者。
- シーン: ターミナルを起動し、その場で Todo を入力・確認する。

## 3. 現状（実装済みの要件）

- ユーザーは Todo の「タイトル」と「本文」を入力して 1 件追加できる。
- 1 件追加するごとに、入力内容（タイトル・本文）が表示される。
- 「続けますか？(y/n)」の確認に対し、`n` 以外を入力すると入力を継続し、
  `n` を入力すると入力を終了する。
- 入力を終了すると、登録した Todo の一覧（タイトル・本文）を表示する。
- データはメモリ上にのみ保持し、アプリを終了すると消える。

## 4. 今後の候補（未確定・草案）

優先度や実施可否はレビューで決定する。

- Todo の「完了 / 未完了」状態の管理。
- Todo の削除・編集。
- 番号付き一覧表示や、一覧からの選択操作。
- ファイルなどへの永続化（再起動後もデータが残る）。
- 不正な入力に対するエラーハンドリングの改善（現状は `unwrap` を使用）。

## 5. 非対象（現時点ではやらないこと）

- GUI / Web インターフェースの提供。
- 複数ユーザー・ネットワーク同期。
- 外部データベースの利用。

## 6. 制約・前提

- 言語は Rust（`edition = "2024"`）。
- 標準ライブラリのみを使用し、追加の依存クレートは導入しない（現状方針）。
- 設計・コーディング規約は `AGENTS.md` および `docs/` 配下のドキュメントに従う。
