# 実装計画

このドキュメントは、現状の実装と今後の進め方を段階的に整理します。

> 注記: 「フェーズ 0」は実装済みの事実です。
> 「フェーズ 1 以降」は未確定の草案で、優先度・採否はレビューで決めてください。

## フェーズ 0: 基盤（実装済み）

- レイヤー分割の確立（`model` / `presentation`、入口は `mod.rs`）。
- `Todo` 構造体の定義（タイトル・本文）。
- CLI での入力・一覧表示の基本フロー（`presentation::cli::run`）。
- ドキュメント整備（`AGENTS.md`、`docs/`、`README.md`、`rust-model-doc` スキル）。
- 単体テストの土台（`Todo` のテスト）と、ビルド/Lint をクリーンに保つ運用。

## フェーズ 1: 操作性の改善（草案）

- 一覧を番号付きで表示する。
- Todo の削除機能（番号を指定して削除）。
- 入力エラー時のハンドリング改善（`unwrap` の見直し）。

想定する変更箇所:
- `presentation/cli.rs`（操作フロー・表示）。
- 必要に応じて `model/todo.rs`（識別子の追加など）。

## フェーズ 2: 状態管理（草案）

- Todo に「完了 / 未完了」状態を追加する。
- 完了/未完了の切り替え操作と、状態に応じた表示。

想定する変更箇所:
- `model/todo.rs`（状態を表すフィールド or 列挙型の追加。model.md 規約に従う）。
- `presentation/cli.rs`（状態の切り替え・表示）。

## フェーズ 3: 永続化（草案）

- アプリ終了後もデータが残るよう、ファイルなどへ保存・読み込みする。
- 保存形式（例: テキスト / JSON）を決める。依存クレートの要否を検討する。

想定する変更箇所:
- データの入出力を担う新しいレイヤー/モジュール（例: `repository`）の追加を検討。
- `model` と `presentation` の責務を保ったまま、保存処理を分離する。

## 進め方の方針

- 1 フェーズ（または 1 機能）ごとにブランチ・PR を分ける。
- 大きな変更は、実装前に方針を提示して合意を得る（`AGENTS.md` の方針に従う）。
- 各変更後に `cargo build` / `cargo test`、必要に応じて `cargo fmt` / `cargo clippy` を実行する。
- 新しいモデルファイルは `src/model/model.md`（`rust-model-doc` スキル）の規約に従う。
