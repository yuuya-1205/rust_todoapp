# AGENTS.md

このファイルは、AI エージェント（GitHub Copilot など）がこのリポジトリで作業するときに
常に従う基本ルールをまとめたものです。詳細な規約は各 Skill や `docs/` に委譲し、
ここには「入口と地図」となる本質的なルールだけを置きます。

## プロジェクト概要

- Rust 製の CLI Todo アプリケーションです。
- 学習・教育を目的としたプロジェクトのため、コードには手厚い日本語コメントが付いています。
  このトーンとスタイルを尊重してください。

## ビルド・実行・テスト

変更を加えたら、**必ず**以下を実行して結果を確認してください。

```sh
cargo build   # ビルド
cargo run     # 実行
cargo test    # テスト
cargo fmt     # フォーマット
cargo clippy  # Lint
```

- コードを変更したら、最低でも `cargo build` と `cargo test` を実行し、
  成功することを確認してから完了とみなします。
- フォーマットと Lint（`cargo fmt` / `cargo clippy`）も可能な限り通してください。

## アーキテクチャ / ディレクトリ構成

レイヤーごとにディレクトリを分け、各ディレクトリの入口を `mod.rs` にします。

```txt
src/
  main.rs          # 起動地点。処理を書きすぎず、各レイヤーの入口を呼ぶだけ
  model/           # データ構造（ドメイン）
    mod.rs
    todo.rs
  presentation/    # ユーザーとの接点（CLI）
    mod.rs
    cli.rs
```

- `main.rs` には詳細な処理を書かず、各レイヤーの入口を読み込んで実行します。
- 各ディレクトリは `mod.rs` をモジュールの入口にします。
- 公開は最小限にし、必要なものだけ `pub` にします。
- 詳細は `docs/architectured.md` を参照してください。

## コーディング規約

- **コメントは日本語**で書きます。既存コメントを勝手に英語へ翻訳しないでください。
- 公開範囲は最小限に保ちます（不要に `pub` を増やさない）。
- **`src/model/*.rs`（構造体を定義するモデルファイル）を作成・編集するときは、
  `src/model/model.md` の規約に従ってください。** この規約は `rust-model-doc` スキル
  （`.github/skills/rust-model-doc/`）にまとまっているので、それに従います。
  - 先頭にモジュールコメント `//!`
  - 構造体・各フィールドにドキュメントコメント `///`
  - 日本語で記述

## ドキュメントの場所（地図）

実装前に、関連する以下のドキュメントを確認してください。

- `docs/requirements.md` … 何を作るか（要件）
- `docs/architectured.md` … モジュール構成・設計
- `docs/cording-rules.md` … コーディングルール
- `docs/review-checklist.md` … レビュー観点
- `src/model/model.md` … モデルファイルの記述規約
- `src/presentation/presentation.md` … プレゼンテーション層の説明

## AI エージェントへの作業指針

- 大きな変更は、いきなり実装せず **まず方針（プラン）を提示** し、合意してから実装します。
- 1つのセッション（作業）では1テーマに集中し、区切りがついたらコミットします。
- 既存の日本語コメント・教育的なトーンを尊重し、急に文体を変えないでください。
- 変更は対象に絞り、無関係なファイルには手を入れないでください。

## やってはいけないこと

- `main` ブランチに直接コミットしないでください（作業ブランチで行う）。
- リクエストに関係のないファイルを変更しないでください。
- 日本語コメントを勝手に英語へ翻訳しないでください。
- model.md の規約と矛盾する独自フォーマットを導入しないでください。
