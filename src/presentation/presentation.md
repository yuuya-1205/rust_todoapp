## ユーザーとの接点

`presentation` は、ユーザーとアプリケーションのやり取りを担当するレイヤーです。

このプロジェクトでは CLI（Command Line Interface）を採用しており、ターミナル上で
ユーザーから入力を受け取り、結果を表示します。

### このレイヤーの責務

- ユーザーに入力を促す（プロンプトの表示）。
- 入力された値を受け取り、`model` のデータ構造（例: `Todo`）に変換する。
- 登録済みの内容を一覧として表示する。
- アプリケーションの操作フロー（繰り返し入力・終了判定など）を管理する。

`model` がデータの「形」を定義するのに対し、`presentation` は「ユーザーとの接点」を
担当します。役割を分けることで、表示や入力方法を変更しても `model` に影響が及びにくく
なります。

### ファイル構成

```txt
presentation/
  mod.rs    # presentation ディレクトリーの入口。cli モジュールを公開する。
  cli.rs    # CLI での操作フローを定義する。
```

#### `mod.rs`

`mod.rs` は `presentation` ディレクトリーの入口ファイルです。

```rust
pub mod cli;
pub use cli::run;
```

`pub mod cli;` で `cli` モジュールを公開し、`pub use cli::run;` で `run` 関数を
`presentation::run()` として呼び出せるように再エクスポートします。これにより、
`main.rs` は `cli` の内部構造を意識せずに入口だけを呼べます。

```rust
// main.rs から
presentation::run();
```

必要なものだけを公開する、という方針に沿って、外部から使う `run` 関数だけを
入口として見せています。

#### `cli.rs`

`cli.rs` は CLI での操作フローを定義するファイルです。

- ユーザーにタイトルと本文の入力を促す。
- 入力された値から `Todo` を生成する。
- 「続けますか？」の確認で、入力を繰り返すか終了するかを判定する。
- 最後に、登録された Todo の一覧を表示する。

CLI は Command Line Interface の略で、ターミナル上でユーザーとやり取りするための
操作部分を表します。
