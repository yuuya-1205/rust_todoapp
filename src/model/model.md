## `<ファイル名>.rs` の構成

`<ファイル名>.rs` は、`<アプリ名>` で扱う `<役割>` を定義するためのファイルです。
このファイルでは、`<構造体名>` という構造体を定義します。  
`<構造体名>` は `<何を表すか>` を表し、`<持っている情報>` を持ちます。

### モジュールコメント

ファイルの先頭には、`//!` を使ってモジュール全体の説明を書きます。

```rust
//! <構造体名>のデータ構造を定義するモジュール。
//!
//! このモジュールは<アプリ名>で扱う<役割>を表す
//! [`<構造体名>`] 構造体を提供します。
```

`//!` は、このファイル全体がどのような役割を持つかを説明するコメントです。

### 構造体コメント

構造体には、`///` を使って説明を書きます。

```rust
/// 1件の<役割>を表す構造体。
///
/// <構造体名>として扱うための情報を持ちます。
pub struct <構造体名> {
    /// <フィールド1>を表します。
    pub <field_1>: String,
    /// <フィールド2>を表します。
    pub <field_2>: String,
}
```

`///` は、構造体やフィールドなど、特定の要素を説明するコメントです。

### 各フィールドの意味

`<構造体名>` は、次のフィールドを持ちます。

- `<field_1>`
  - `<field_1>` の意味を説明します。
- `<field_2>`
  - `<field_2>` の意味を説明します。

### 完成形

```rust
//! <構造体名>のデータ構造を定義するモジュール。
//!
//! このモジュールは<アプリ名>で扱う<役割>を表す
//! [`<構造体名>`] 構造体を提供します。
/// 1件の<役割>を表す構造体。
///
/// <構造体名>として扱うための情報を持ちます。
pub struct <構造体名> {
    /// <フィールド1>を表します。
    pub <field_1>: String,
    /// <フィールド2>を表します。
    pub <field_2>: String,
}
```
